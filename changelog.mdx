## Recent Updates

Stay informed about PRCS enhancements. You access the latest version automatically upon login. Review release notes below to understand new capabilities and resolved issues.

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added Slack integration for real-time notifications on service requests.
- Introduced employee directory search with advanced filters for departments and roles.
- Enabled multi-factor authentication (MFA) setup during registration.

## Improvements

- Optimized dashboard load time by `>30%` for better performance.
- Enhanced email verification flow with one-click resend options.

## Bug Fixes

- Fixed intermittent login failures during peak hours.
- Resolved issue where service request status updates did not reflect in real-time.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- Launched service request ticketing system with priority levels.
- Integrated Google Workspace for seamless calendar-based service bookings.

## Bug Fixes

- Corrected registration form validation for international email domains.
- Patched security vulnerability in session token handling.

## Improvements

- Updated UI components to match brand color `#5f6667` for consistency.

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]}>

## Initial Release

- Core employee sign-in and registration using email.
- Basic service catalog access for corporate resources.
- Dashboard overview of available employee services.

## Breaking Changes

- Migrated to new authentication endpoint. Update your integrations to use `https://api.example.com/v1/auth`.

</Update>

## Upgrade Instructions

Follow these steps to ensure your PRCS instance uses the latest version.

<Steps>
  <Step title="Check Current Version" icon="check-circle">

    Log in to your dashboard and navigate to **Settings > About**.

    ```bash
    curl -H "Authorization: Bearer YOUR_TOKEN" \
         https://api.example.com/v1/version
    ```

  </Step>
  <Step title="Backup Data" icon="database">

    Export your service requests and user data.

  </Step>
  <Step title="Apply Update" icon="download">

    PRCS updates automatically. Refresh your browser after login.

  </Step>
</Steps>

<Callout kind="tip">

  Monitor the [status page](https://status.example.com) during updates for any downtime notifications.

</Callout>
